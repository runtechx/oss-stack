# Open First

A **RunTech** adota uma política **“Open First”**, priorizando o uso, promoção e contribuição para **software de código aberto (Open Source)** e **padrões abertos** em todas as suas decisões tecnológicas.

Esta abordagem permite construir uma infraestrutura mais transparente, auditável e sustentável, reduzindo dependências de fornecedores e garantindo maior controlo sobre os sistemas críticos. O objetivo é assegurar soluções flexíveis, seguras e alinhadas com as necessidades reais do negócio.

## Open First Policy

Na **RunTech**, a adoção de tecnologia segue um modelo simples e intencional:

1. **Preferir, utilizar e promover software de código aberto**
   O uso de soluções open source é a primeira escolha por defeito. Sempre que possível, privilegiamos ferramentas self-hosted, incentivando também a sua evolução, adoção interna e contributo para a comunidade. Esta abordagem reforça a independência tecnológica, a transparência e a eficiência operacional.

2. **Adotar alternativas proprietárias apenas quando necessário**
   Quando não existir uma solução open source viável, recorremos a software proprietário ou SaaS de forma consciente, privilegiando fornecedores alinhados com standards abertos, boas práticas de segurança e privacidade. Estas decisões são revistas periodicamente, com o objetivo de migrar para alternativas open source sempre que possível.

## Princípios Operacionais

* **Open First by Design** – O open source é o ponto de partida para qualquer decisão
* **Automação** – Infraestrutura gerida como código sempre que possível
* **Segurança** – Implementação de controlos desde a base (security by design)
* **Escalabilidade** – Preparado para crescimento e evolução contínua
* **Observabilidade** – Monitorização e visibilidade completa dos sistemas
  
## Contribuições

Acreditamos que o ecossistema open source evolui com a colaboração.

Caso pretenda sugerir novas tecnologias, melhorias ou alternativas às soluções atualmente adotadas, convidamos à abertura de um issue neste repositório.

Todas as contribuições são bem-vindas, especialmente quando:
* Apresentam alternativas open source alinhadas com os nossos princípios
* Incluem justificação técnica e/ou comparação com soluções existentes
* Contribuem para maior eficiência, segurança ou escalabilidade
